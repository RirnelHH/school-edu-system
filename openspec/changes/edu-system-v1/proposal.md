## Why

技工院校需要一个统一的教务管理平台，整合人才培养计划、排课、成绩管理、请假审批等核心业务流程。当前依赖人工和纸质流程，效率低、易出错。

## What Changes

### 核心功能模块

1. **账号与认证系统**
   - 多角色账号（学生/教师/班主任/教研组长/主任/管理员）
   - JWT认证 + 微信OAuth登录
   - 账号安全策略（登录锁定、密码过期）

2. **基础数据管理**
   - 学校/院系/专业/班级/学生/教师数据
   - Excel批量导入

3. **人才培养计划**
   - 教研组长制定 → 主任审批

4. **排课系统**
   - 课时配置（理论/上机比例）
   - 教师时间偏好
   - Excel课表模板导入
   - 机房管理和冲突检测
   - 课表版本管理

5. **授课计划与教案**
   - 教师制定授课计划
   - 教案提交 → 教研组长审核 → 主任审批

6. **成绩管理**
   - 成绩录入（单条/批量Excel导入）
   - 教研组长审核 → 发布
   - 学生查询本人成绩

7. **教材管理**
   - 教材库维护
   - 订购申请 → 审批 → 入库

8. **班级管理**
   - 学生职务设定（班长/学习委员等）
   - 班主任分配/更换

9. **请假审批**
   - 学生申请 → 班主任审批 → 主任审批

10. **每日签到**
    - GPS定位签到
    - 迟到判断
    - 班主任查看出勤率

11. **任务与作业**
    - 教师发布任务
    - 学生提交
    - 交稿状态追踪（已交/未交/迟到）

12. **教师课时统计**
    - 重复课时系数（人数最多班级=1.0，其他按配置）
    - 班级人数系数
    - 停课扣减

13. **退学处理**
    - 班主任发起 → 主任审批
    - DOCX表格自动预填学生信息

## Capabilities

### New Capabilities

- `account-auth`: 账号认证（JWT/微信OAuth）
- `user-roles`: 多角色系统（兼任角色）
- `base-data`: 基础数据管理（学校/专业/班级/学生/教师）
- `talent-plan`: 人才培养计划
- `course-management`: 课程与课标管理
- `scheduling`: 排课系统（机房/模板/偏好）
- `teaching-plan`: 授课计划与教案
- `grade-management`: 成绩管理
- `textbook-management`: 教材管理
- `class-management`: 班级管理（含职务）
- `leave-approval`: 请假审批流
- `daily-checkin`: 每日签到（GPS）
- `task-homework`: 任务与作业
- `teaching-hours`: 教师课时统计
- `withdrawal`: 退学处理

### Modified Capabilities

（暂无）
