# 排课系统逻辑修复

## 变更类型
功能修复 + 业务规则补充

## 变更原因
原有自动排课逻辑不符合实际业务规则：
1. 原代码按"单节"排课，实际需要按"两节一起"排
2. 冲突检测信息不完整（课程名显示"待查"）
3. 同一时段教师/班级限制逻辑虽有但数据不完整

## 新增/修改的业务规则

### 排课节次规则
- 每节课必须两节一起排
- 有效节次对：(1,2), (3,4), (5,6), (7,8)
- 代码中用奇数 1/3/5/7 表示起始节，end 自动加 1

### 冲突检测规则
| 冲突类型 | 条件 |
|----------|------|
| 教室冲突 | 同一时段（weekday+period）同一教室已被占用 |
| 教师冲突 | 同一时段（weekday+period）同一教师已被安排到任何班级 |
| 班级冲突 | 同一时段（weekday+period）同一班级已有其他课程 |

### 冲突信息完整性
- 返回真实课程名（查询 course 表）
- 返回真实教师名（查询 user 表）
- 返回真实班级名（查询 class 表）

## 技术修改

### scheduling.service.ts
1. `checkConflicts()` - 补充真实名称查询
2. `autoSchedule()` - 修改节次逻辑，按 (1,2)(3,4)(5,6)(7,8) 配对排课

## 影响范围
- 自动排课算法
- 冲突检测返回信息
